# Phase 1 — UI Design Contract: Tab + Formulario Base

> Visual and interaction contract for Phase 1 of Galdi Gestión Tab 5 "Presupuestos".
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Governing constraint:** The new tab must be visually indistinguishable from existing tabs 1–4.
> Every token, pattern, and component in this spec is derived from the existing `public/gestion/index.html`.

---

| Property | Value |

## Spacing Scale

All values replicate the existing inline spacing used in tabs 1–4 of `index.html`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | `margin-bottom: 4px` on `<label>` elements |
| sm | 8px | Button padding vertical, gap between inline elements |
| md | 14px | `margin-bottom: 14px` between grid rows and card internals |
| lg | 22px | `.card` internal padding (22px) |
| xl | 28px | `.panel` padding (28px top/sides) |
| 2xl | 18px | `margin-bottom: 18px` between `.card` elements |
| gap-sm | 10px | Flex gap between action buttons |
| gap-md | 14px | `.grid-2` and `.grid-3` gap |

**Exceptions — inherited from existing index.html stylesheet (do NOT change):**
- md (14px): `.grid-2` gap and `.card` internal spacing — inherited, pre-existing
- lg (22px): `.card` padding — inherited, pre-existing
- 2xl (18px): `.card` margin-bottom — inherited, pre-existing
- gap-sm (10px): button flex gap — inherited, pre-existing
- Input padding 8px 11px: existing input style — inherited, pre-existing

None of these values conform to the 4-point grid because the pre-existing system predates this grid convention. All are verbatim measurements from `public/gestion/index.html` and MUST NOT be changed to preserve visual parity with tabs 1–4.

## Component Inventory

These are the exact CSS class patterns to reuse from the existing stylesheet. No new classes unless a gap exists.

### Tab nav entry (Tab 5)

```html
<div class="tab" onclick="showTab('presupuestos',this)">5 · Presupuestos</div>
```

- Same `.tab` class: `padding: 13px 24px`, `font-size: 11px`, `letter-spacing: .14em`, `text-transform: uppercase`
- Active state: `color: var(--gold2)`, `border-bottom: 2px solid var(--gold)`, `font-weight: 600`, `background: var(--cream)`

### Panel container

```html
<div id="tab-presupuestos" class="panel">
  <!-- cards go here -->
</div>
```

- `.panel`: `display: none; padding: 28px; max-width: 960px; margin: 0 auto`
- `.panel.active`: `display: block`

### Card

```html
<div class="card">
  <div class="card-title">Título de sección</div>
  <!-- content -->
</div>
```

- `.card`: `background: #fff; border: 1px solid var(--border); border-radius: 8px; padding: 22px; margin-bottom: 18px`
- `.card-title`: `font-size: 10px; letter-spacing: .2em; text-transform: uppercase; color: var(--gold); margin-bottom: 14px; font-weight: 600`

### Grid layout

- `.grid-2`: `display: grid; grid-template-columns: 1fr 1fr; gap: 14px`
- `.grid-3`: `display: grid; grid-template-columns: 1fr 1fr 1fr; gap: 14px`
- Mobile (`max-width: 600px`): both collapse to `grid-template-columns: 1fr`

### Input / select

```html
<div>
  <label>Nombre del cliente *</label>
  <input type="text" id="pres-nombre" placeholder="ej: María González">
</div>
```

- `label`: `font-size: 11px; color: var(--muted); letter-spacing: .06em; display: block; margin-bottom: 4px`
- `input, select`: `width: 100%; padding: 8px 11px; border: 1px solid var(--border); border-radius: 6px; font-family: 'Jost',sans-serif; font-size: 13px; color: var(--brown); background: var(--cream)`
- Focus: `border-color: var(--gold); outline: none`

### Read-only field (COT-GXXX)

```html
<div>
  <label>N° Presupuesto</label>
  <input type="text" id="pres-numero" readonly
         style="background: var(--sand); color: var(--gold2); font-weight: 600; letter-spacing: 0.08em; cursor: default;">
</div>
```

- Background: `var(--sand)` — same as table headers and tab bar (visually "locked")
- Text color: `var(--gold2)` — distinguishes it as a system-generated value
- Font-weight: 600 — visually heavier to signal it is the document identifier
- No border highlight on focus (add `pointer-events: none` or `readonly` attr)
- No badge/pill — plain input with distinct background is sufficient and consistent with existing patterns

### Textarea (observaciones)

```html
<div>
  <label>Observaciones / Notas</label>
  <textarea id="pres-obs" rows="3"
    style="width:100%;padding:8px 11px;border:1px solid var(--border);border-radius:6px;
           font-family:'Jost',sans-serif;font-size:13px;color:var(--brown);
           background:var(--cream);resize:vertical;"></textarea>
</div>
```

- Inherits all input styles; adds `resize: vertical` only
- Focus: same `border-color: var(--gold)` as all other inputs

### Buttons (Phase 1 — placeholder state)

Phase 3 will implement the functional "Guardar borrador" and "Generar presupuesto" buttons. In Phase 1, render them as **visually present but disabled** so the form feels complete:

```html
<div style="display:flex;gap:10px;margin-top:18px;justify-content:flex-end;flex-wrap:wrap;">
  <button class="btn btn-gray" disabled
          style="opacity:0.5;cursor:not-allowed;">Guardar borrador</button>
  <button class="btn" disabled
          style="opacity:0.5;cursor:not-allowed;">Generar presupuesto</button>
</div>
```

- `.btn`: gold background — "Generar presupuesto" is the primary action (implemented in Phase 3)
- `.btn-gray`: muted background — "Guardar borrador" is secondary
- `disabled` + `opacity: 0.5` + `cursor: not-allowed` — clearly inactive without removing them from DOM
- Buttons become fully functional in Phase 3; no click handlers in Phase 1

## COT-GXXX Number Logic Contract

- On tab open: query Firestore `galdi_presupuestos` collection, find maximum existing `numero` field
- Parse last 3 digits: `COT-G077` → integer `77`
- Next number = max + 1, zero-padded to 3 digits: `COT-G078`
- If collection is empty or query fails: default to `COT-G077`
- Value is pre-filled into `#pres-numero` as read-only
- Prefix is always `COT-G` — never changes
- Displayed exactly as stored: `COT-G077`, `COT-G100`, `COT-G999`
